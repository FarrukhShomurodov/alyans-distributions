# Формат ответа СОВА API при создании заказа

Эндпоинт: `POST {SOVA_API_URL}/order`
Заголовок: `Authorization: Bearer {SOVA_API_TOKEN}`

ALYANS DISTRIBUTIONS отправляет **по одному заказу за раз** и ждёт ответ перед отправкой следующего. Если заказ получил ошибку — он остаётся непомеченным (`sova_exported_at = null`) и будет переотправлен через 30 минут планировщиком.

---

## 1. Успешный ответ

Любой из вариантов будет воспринят как успех:

### Вариант A — минимальный
```json
{
  "status": "ok"
}
```

### Вариант B — с подтверждением id
```json
{
  "status": "ok",
  "order_id": "55",
  "sova_order_id": 12345,
  "message": "Заказ принят"
}
```

### Вариант C — success: true
```json
{
  "success": true,
  "order_id": "55",
  "sova_order_id": 12345
}
```

---

## 2. Ответ с ошибкой

Любой из вариантов будет воспринят как ошибка. **Заказ НЕ будет помечен как отправленный и повторится через 30 мин.**

### Вариант A — через status
```json
{
  "status": "error",
  "message": "Товар с nmatr_id=118 не найден в каталоге",
  "order_id": "55"
}
```

### Вариант B — через success: false
```json
{
  "success": false,
  "error": "Недостаточно остатков на складе",
  "order_id": "55"
}
```

### Вариант C — через result: error
```json
{
  "result": "error",
  "description": "Клиент с таким телефоном заблокирован",
  "order_id": "55"
}
```

### Вариант D — через массив errors (если заказов несколько)
```json
{
  "errors": [
    {
      "order_id": "55",
      "code": "PRODUCT_NOT_FOUND",
      "message": "Товар nmatr_id=118 не найден"
    }
  ]
}
```

---

## 3. Типовые ошибки, которые стоит отдавать

| Код ошибки                | Ситуация                                                      |
|---------------------------|---------------------------------------------------------------|
| `PRODUCT_NOT_FOUND`       | `nmatr_id` отсутствует в каталоге СОВЫ                        |
| `INSUFFICIENT_STOCK`      | Товар есть, но остатков меньше чем заказано                   |
| `INVALID_PAYLOAD`         | Некорректный формат данных (отсутствует обязательное поле)    |
| `DUPLICATE_ORDER`         | Заказ с таким `order_id` уже существует в СОВЕ                |
| `INVALID_PRICE`           | Цена не совпадает с ценой в СОВЕ                              |
| `CUSTOMER_BLOCKED`        | Клиент в чёрном списке                                        |
| `INVALID_DELIVERY`        | Некорректные данные о доставке                                |
| `INTERNAL_ERROR`          | Внутренняя ошибка СОВЫ                                        |

## 5. HTTP-коды

- `200 OK` — Сова приняла запрос (даже если в теле статус "error" — заказ обработан, просто с ошибкой бизнес-уровня)
- `4xx` / `5xx` — транспортная ошибка, ALYANS DISTRIBUTIONS воспринимает как "нет ответа" и повторит позже

---

## 6. Рекомендации

1. **Идемпотентность**: если ALYANS DISTRIBUTIONS отправит один и тот же `order_id` повторно — вернуть успех с пометкой "дубликат", а не создавать новый заказ. Это обработка ретраев.

2. **Быстрый ответ**: СОВА должна отвечать в пределах 30 секунд (таймаут клиента).

3. **Чёткая ошибка**: в `message` давайте полезный текст для менеджера — например, «Товар 'Жидкость Bubble' (nmatr_id=118) заблокирован», а не просто «Error».
